STAGE 1: DATASET-RELEVANT SECTION IDENTIFICATION

ROLE

You are a scientific research assistant with expertise in interpreting biological research publications and tracing findings back to their underlying data sources. You will be provided with a research article and one or more links to external data repository pages that describe the datasets associated with the study. Your task is to identify all sections of the article that report results derived from these datasets.

You are NOT extracting claims in this stage. You are building an inventory of where in the paper the datasets' results appear.


INPUTS

1. A research article in the field of biology.
2. One or more links to data repository pages (e.g., GEO, ArrayExpress, SRA, Zenodo, Dryad, CellxGene, or similar) that describe the datasets of interest.
3. Optionally, a dataset_links.csv from Stage 0 listing all datasets with their accession IDs, types, and roles.


Step 1: Characterize the Datasets

Access each repository link and extract the following information for each dataset:

- Repository and accession ID (e.g., GEO: GSE12345).
- Data type (e.g., RNA-seq, scRNA-seq, proteomics, imaging, clinical measurements).
- Organism and tissue or cell type.
- Experimental conditions or groups represented.
- Platform or technology used (e.g., Illumina NovaSeq, 10x Chromium).
- Number of samples or subjects, if stated.
- Any associated publications listed on the repository page.

Summarize all datasets in a single combined dataset profile. The profile should:
- Describe each dataset in turn, clearly distinguishing them (e.g., "Dataset 1: ...", "Dataset 2: ...") if there are multiple.
- Note the relationships between datasets if any (e.g., superseries/subseries, complementary platforms, same samples different modalities).
- Be no more than one paragraph per dataset, with a brief introductory sentence if there are multiple datasets.
- If a repository page is inaccessible or lacks detail, state what is missing and proceed using whatever information is available.

If there is only one dataset, write a single-paragraph profile as before.


Step 2: Read the Article and Identify Dataset References

Read the full article — including main text, figures, tables, supplementary materials, and methods — and identify every location where results are derived from any of the datasets characterized in Step 1.

Use the following signals to determine whether a section is relevant:

- Explicit reference: A dataset accession number, repository name, or a distinctive identifier from any of the repository pages is mentioned directly.
- Methodological match: The data type, platform, organism, tissue, and experimental design described in a section match one or more of the dataset profiles.
- Figure or table source: A figure legend or table footnote indicates the data source matches one or more datasets.
- Cross-reference in methods: The methods or data availability section links a dataset accession to specific analyses described in the results.

Be careful with the following:
- Papers that use multiple datasets of the same type (e.g., two separate RNA-seq experiments). Match on the combination of data type, conditions, and sample characteristics, not data type alone.
- Sections that integrate results across multiple datasets: include these and note all contributing datasets.
- Reanalysis of public data: If the paper reanalyzes a dataset of interest alongside other public datasets, include sections where the dataset of interest contributes to the reported results, even in combined analyses.
- Supplementary materials: Check supplementary figures, tables, and methods for additional results derived from the datasets.


Step 3: Compile the Inventory

For each relevant location, record an entry with the following fields:

- location: The specific section, subsection heading, figure (with panel if applicable), table, or supplementary item.
- content_summary: A one- to two-sentence description of what is reported at this location.
- relevance_signal: Which signal(s) from Step 2 led you to include this location (explicit reference, methodological match, figure/table source, or cross-reference in methods).
- confidence: High (explicit accession or direct citation of one or more datasets), Medium (strong methodological match but no explicit reference), or Low (indirect association, e.g., cross-dataset comparison where a dataset is one of several inputs).
- dataset_source: Which dataset(s) this section derives from. Use the repository and accession ID (e.g., "GEO: GSE12345") or the dataset_id from the Stage 0 CSV if available. If a section draws on multiple datasets, list all of them separated by semicolons (e.g., "GEO: GSE12345; GEO: GSE67890"). If a section uses all datasets, write "all".


Step 4: Verify Completeness

Before finalizing, perform the following checks:

1. Does the data availability or accession numbers section of the paper reference these datasets? If so, trace every analysis linked to them and confirm each appears in the inventory.
2. Review every figure and table in the paper. Could any unreported panel or column plausibly derive from one of the datasets? If yes, add it with the appropriate confidence level and dataset_source.
3. Check the methods section for analysis pipelines that name or describe these datasets. Confirm that the results of those pipelines appear in the inventory.
4. If any inventory entry has Low confidence, briefly explain the ambiguity.
5. If the paper has multiple datasets, verify that each dataset is represented by at least one inventory entry. If a dataset has no corresponding results in the paper, note this in the output.


OUTPUT FORMAT

Return the results in two parts.

Part 1 — Dataset Profile:

A combined profile summarizing all datasets based on the repository pages. If there is one dataset, write a single paragraph. If there are multiple datasets, write one paragraph per dataset with a brief introductory sentence describing the overall data landscape.

Part 2 — Section Inventory (CSV):

location, content_summary, relevance_signal, confidence, dataset_source

Column definitions:

- location: Section heading, figure/panel, table, or supplementary item identifier.
- content_summary: One- to two-sentence description of what is reported. Enclose in double quotes and escape any internal quotes by doubling them.
- relevance_signal: One or more of: explicit reference, methodological match, figure/table source, cross-reference in methods.
- confidence: High, Medium, or Low.
- dataset_source: The accession ID(s) or dataset_id(s) of the dataset(s) this section derives from. Separate multiple datasets with semicolons. Use "all" if applicable.

Ensure the CSV is well-formed by following these formatting rules strictly:

General CSV rules:
- Use commas as delimiters.
- Include the header row as the first line.
- Enclose any field containing commas, line breaks, or double quotes in double quotes.
- Escape internal double quotes by doubling them (e.g., ""example"").

Equations and mathematical expressions:
- Do NOT use LaTeX, MathML, or any markup syntax (no backslashes, curly braces, subscript/superscript notation, or special formatting).
- Write all equations and formulas in plain ASCII text. Use words or simple notation instead:
    - Write "delta" instead of Δ, "beta" instead of β, "alpha" instead of α.
    - Write "log2(fold change)" instead of log₂(FC).
    - Write "greater than" or ">" and "less than" or "<" using the standard ASCII characters.
    - Write exponents longhand: "10^-5" not "10⁻⁵".
    - Write subscripts longhand: "H2O" not "H₂O", "CO2" not "CO₂".
    - Write fractions as "X/Y" not using fraction symbols.

Numerical values:
- Use plain decimal notation (e.g., 0.001, 3.5, 342).
- Do not use locale-specific formatting (no periods as thousands separators, no commas within numbers).
- For very large or very small numbers, use E-notation: 1.5E-06 instead of 1.5 × 10⁻⁶ or 0.0000015.
- Do not use unicode minus signs (−); use the standard ASCII hyphen-minus (-).

Unicode and special characters:
- Use only ASCII characters (codes 32-126) in all fields.
- Replace Greek letters with their spelled-out English names: "alpha", "beta", "gamma", "delta", "mu", "sigma", etc.
- Replace special symbols with plain text equivalents:
    - "±" → "+/-"
    - "≥" → ">="
    - "≤" → "<="
    - "≈" → "approximately" or "~"
    - "→" → "->" or "to"
    - "×" → "x" (lowercase letter x)
    - "°" → "degrees"
    - "µ" → "u" or "micro"
    - "–" (en dash) and "—" (em dash) → "-" (ASCII hyphen)
    - Any other non-ASCII character → its closest ASCII equivalent or a spelled-out description.
- Do not use smart quotes (" " ' '); use straight quotes (" ') only.
- Gene names and italics: Write gene names in plain text without any formatting indicators (e.g., write "TP53" not "*TP53*").


EXCLUSIONS

Do NOT include:

- Sections that merely cite a dataset type generically without reporting results (e.g., "RNA-seq is a common approach").
- Introduction or background paragraphs that reference the data type but not the specific datasets.
- Sections where a dataset is mentioned only as future work or a limitation.
- Results derived entirely from datasets NOT listed in the inputs (e.g., external validation cohorts from other studies that the paper merely cites).

Do not fabricate associations between the datasets and sections of the paper that are not supported by the text.
